OVERVIEW

Application works by making HTTP GET request to the following url
`http://apps.lsim.az/rbt/app?&user=USER&pass=PASSWORD&mod=MODE&cmd=COMMAND[&param1=val1...]`.
Response will be in xml format. For all responses there is `responsecode`, where will be the
result of command. Beside that there can be additional parameters for data. Definitions of the
parameters are as follows.
`user` - user name (request from admin while using this api)
`pass` - user password (request from admin while using this api)
`mod` - which api mode to use
`cmd` - which command of the mode to execute
These four parameters are mandatory. There can be extra paramaters, control parameters from now,
to control the resulting output.


MODES

There are four modes - `content`, `category`, `statistics`, `shortcode` and `tone`. Every mode
has its own commands.
`content` - all procedures related to zengimcell contents
`category` - all procedures related to zengimcell categorys
`shortcode` - all procedures related to zengimcell short codes
`tone` - all procedures related to zengimcell tones provided by operator
`statistics` - all procedures related to statistics (load logs)


COMMANDS

 `CONTENT`

  `ADD` (only for admin)
  This command is used to add new content. Below are the parameters for this command.
  `category_id` - id of category to which the content belongs (use `catalog`
                  command of `category` mode to get ids)
  `code` - content code provided by operator
  `artist` - artist name
  `song` - song name
  `pid` - partner exclusive content, leave empty for all partners
  `visible` - visible or not

  `UPDATE` (only for admin)
  This command is used to update content data. Below are the parameters for this command.
  `id` - id of content (use `catalog` command to get the ids)
  `category_id` - id of category to which the content belongs (use `catalog`
                  command of `category` mode to get ids)
  `code` - content code provided by operator
  `artist` - artist name
  `song` - song name
  `pid` - partner exclusive content, leave empty for all partners
  `visible` - visible or not

  `DELETE` (only for admin)
  This command is used to delete content. Below are the parameters for this command.
  `id` - id of content (use `catalog` command to get the ids)

  `GET`
  This command is used to get the details of content by its id. To get the id the content
  use `catalog` command. Below are the parameters for this command.
  `id` - id of the content
  `pid` - include partner exclusive content (only for admin)

  `CATALOG`
  This command is used to get the whole list of contents. There is parameters that can control
  how the result will be sorted. There is also a search parameter and a control parameter to
  define where to search. The result will be in pages. Number of contents can be controlled
  by control parameter. To navigate by pages use page control parameter. Below are the
  parameters for this command.
  `sort` - define sorting
           4 - sort by artist name
           5 - sort by added date (descending)
           6 - sort by song name
           7 - sort by code (default)
           8 - sort by id (asc)
           9 - sort by id (desc)
  `category_id` - id of category (use `catalog` command of `category` mode to get ids)
  `pid` - the partner id (only for admin)
  `ov` - only visibles (only for admin)
  `page` - page number to show
  `num_per_page` - number of contents per page (default is 10)
  `search_string` - the string to search
  `search` - where to search
             1 - search by artist name
             2 - search by song name
             3 - search by code
             4 - search by all above

  `TOP`
  This command is used to get the top downloaded contents. By setting control parameter
  number of contents can be controlled. Below are the parameters for this command.
  `num` - number of top content
  `pid` - include partner exclusive content (only for admin)

  `LAST`
  This command is used to get the last added contents. By setting control parameter number
  of contents can be controlled. Below are the parameters for this command.
  `num` - number of top content
  `pid` - include partner exclusive content (only for admin)


 `CATEGORY`

  `ADD` (only for admin)
  This command is used to add new category. Below are the parameters for this command.
  `name` - category name
  `position` - category position
  `visible` - visible or not

  `UPDATE` (only for admin)
  This command is used to update category data. Below are the parameters for this command.
  `id` - id of category (use `catalog` command to get the ids)
  `name` - category name
  `position` - category position
  `visible` - visible or not

  `DELETE` (only for admin)
  This command is used to delete category and all its contents. Below are the parameters for this command.
  `id` - id of category (use `catalog` command to get the ids)

  `GET`
  This command is used to get the details of category by its id. To get the id the category
  use `catalog` command. Below are the parameters for this command.
  `id` - id of the category

  `CATALOG`
  This command is used to get the whole list of categories. There is parameters that can control
  how the result will be sorted. Below are the parameters for this command.
  `sort` - define sorting
           1 - sort by category name
           2 - sort by position (default)
           3 - sort by added date (descending)

 `TONE`

  `SUBSCRIBE`
  This command is used to load the tone to subscriber. Below are the parameters for this command.
  `msisdn` - subscribers number
  `tone` - content code provided by operator

 `SHORTCODE` (only for admin)

 `STATISTICS`

  `UDPATE` (only for admin)
  This command is used to update the logs from operator provided interface.

  `GET`
  This command is used to get logs. By setting control parameters the resulting output can be
  controlled. Result will be in pages. Below are the parameters for this command.
  `from` - logs from given date (yyyy-mm-dd hh:mi:ss)
  `to` - log to date, given date not included (yyyy-mm-dd hh:mi:ss)
  `msisdn` - subscribers msisdn
  `pid` - only given partner logs (only for admin)
  `code` - search by code provided by operator
  `amount` - search by amount
  `page` - page number to show
  `num_per_page` - number of contents per page (default is 10)

RESPONSE CODES

0 - Success
1 - System error (try later)
2 - Duplicate position given error
3 - Update error
4 - Invalid category id error
5 - Null value error
6 - mod is wrong.Accepted modes (content,category,shortcode,tone,statistics)
7 - Access denied - username or password is wrong
8 - The number exists on comverse rbt
9 - Subscribtion already exists
10 - The rbt does not exist
11 - The coding format error
12 - Msisdn does not exist
13 - Balance is insufficient
14 - Msisdn is not specified
15 - The code is not specified
16 - Tone code doesnt belong to originator
17 - This tone already exists
18 - Msisdn format error
19 - Tone code does not exist
20 - Internal rbt gateway error
21 - Msisdn format error
